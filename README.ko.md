# ContestRedbook

[中文版本](./README.zh.md) | [홈 README](./README.md)

공모전 정보를 바로 샤오홍슈 게시물 형태로 가공해 주는 경량 프런트엔드 도구다.

이 프로젝트는 일반적인 관리형 SaaS가 아니라, 매우 구체적인 콘텐츠 제작 워크플로를 겨냥한 도구다. 사용자는 공모전 정보를 입력하거나 공식 사이트 링크를 넣고, 최종적으로 4장의 카드 이미지, 게시용 캡션, 텍스트 요약본까지 한 번에 얻을 수 있다. DeepSeek를 연결하면 공식 사이트 내용 추출, 스타일 방향 생성, 비주얼 스타일 생성도 자동화된다.

결과물 기준으로 보면 “공모전 정보 패키징 도구”이고, 구현 기준으로 보면 완성도 높은 단일 HTML Web App이다.

---

## 이 프로젝트가 해결하는 문제

공모전 정보는 대개 공식 사이트 안에 흩어져 있다. 실제로 콘텐츠를 발행하려면 보통 다음 과정을 반복한다.

1. 공식 사이트를 직접 읽는다
2. 공모전명, 일정, 자격, 시상, 주제를 정리한다
3. 게시물용 문장 구조로 다시 쓴다
4. 커버 이미지, 정보 카드, 스타일 추천 카드, 시상 카드를 만든다
5. 최종 이미지를 내보낸다

`ContestRedbook`은 이 과정을 거의 한 번에 묶는다.

사용자가 페이지에 정보를 넣으면 다음 결과를 바로 얻을 수 있다.

1. 커버 이미지
2. 공모전 정보 카드
3. 스타일 방향 추천 카드
4. 시상 상세 카드
5. 복사 가능한 게시물 캡션
6. 텍스트형 공모전 요약

대상 사용자는 넓지 않지만 명확하다. 디자인 공모전 정보를 정리해 콘텐츠로 발행하는 크리에이터, 운영자, 컨설턴트에게 적합하다.

## 핵심 기능 분석

### 1. 구조화된 공모전 입력 폼

입력 폼은 단순하지 않다. 다음 항목을 하나의 데이터 모델로 묶고 있다.

1. 현지어 / 영문 공모전명
2. 주최 기관
3. 참가 자격
4. 주제와 슬로건
5. 마감일과 발표일
6. 제출물
7. 키워드
8. 수상 가치 점수
9. 주제 해설
10. 스타일 방향
11. 시상 구성과 추가 혜택

즉, 이 도구는 먼저 “정보를 구조화”하고, 그 뒤에 카드와 문안을 생성한다.

### 2. 공식 사이트 자동 읽기 + AI 구조화

수동 입력만 있는 도구가 아니다. 공식 사이트 링크를 넣으면 다음 순서로 자동화를 시도한다.

1. 사이트 원문 직접 읽기
2. 실패 시 `r.jina.ai`로 Markdown 변환
3. 본문을 DeepSeek에 전달
4. 구조화된 JSON 응답 받기
5. 폼 자동 입력

이 방식은 사이트별 크롤러를 직접 만드는 것보다 현실적이다. 비정형 공모전 사이트에도 비교적 잘 대응할 수 있다.

### 3. AI 기반 스타일 방향 생성

이 프로젝트는 사실 추출만 하지 않는다. 공모전 주제를 바탕으로 작품 스타일 방향까지 생성한다.

기본적으로 다음 성격을 고르게 포함하도록 설계되어 있다.

1. 전위적 실험
2. 문화 서사
3. 자연 / 재료
4. 상업적 착용성 또는 인터랙션

정보 정리 도구에서 한 단계 더 나아가, 콘텐츠 기획 보조 도구 역할까지 수행한다.

### 4. AI 기반 비주얼 스타일 생성

문안뿐 아니라 시각 시스템도 자동화된다.

1. 메인 컬러와 보조 컬러 생성
2. 밝은 배경 / 어두운 배경 세트 생성
3. 레이아웃 선택
4. 커버 템플릿 선택

동일한 형식의 콘텐츠를 반복 제작할 때, 결과물의 시각적 반복을 줄이는 데 유효하다.

### 5. 4장 카드 템플릿 출력

최종 결과는 4장으로 나뉜다.

1. 커버: 클릭 유도
2. 정보 카드: 실용 정보 전달
3. 스타일 카드: 관점과 아이디어 제공
4. 시상 카드: 참가 가치 강조

이 분할은 단순한 정보 나열이 아니라 실제 소셜 게시 구조를 의식한 설계다.

### 6. `html2canvas` 기반 PNG 내보내기

카드 DOM을 그대로 PNG로 내보낸다. 현재 카드만 저장할 수도 있고 전체 4장을 한 번에 저장할 수도 있다. 별도 디자인 툴로 다시 캡처할 필요가 없다.

## 이 프로젝트가 좋은 개인 프로젝트인 이유

### 1. 문제 정의가 작고 정확하다

너무 넓은 문제를 잡지 않았다. “공모전 사이트 내용을 빠르게 게시 가능한 카드와 문안으로 바꾸기”라는 매우 구체적인 문제를 다룬다. 이런 프로젝트가 실제로 더 잘 작동한다.

### 2. AI를 적절한 위치에 배치했다

AI는 여기서 세 가지 역할에 집중한다.

1. 정보 추출기
2. 스타일 기획 보조기
3. 시각 방향 생성기

즉, AI를 시스템 전체에 억지로 끼워 넣은 것이 아니라 사람이 반복하기 귀찮고 구조화하기 좋은 구간에만 사용했다.

### 3. 결과물이 분명하다

많은 도구는 “정리”에서 끝나지만, 실제 사용자는 “게시 가능한 결과물”을 원한다. 이 프로젝트는 입력에서 이미지와 캡션 출력까지 한 흐름으로 닫힌다.

## 구현 관점의 특징

이 프로젝트의 흐름은 대략 다음과 같다.

```text
공식 사이트 / 수동 입력
        ↓
구조화된 폼 데이터
        ↓
DeepSeek 추출 / 생성
        ↓
공통 데이터 필드
        ↓
4장 카드 렌더링
        ↓
캡션 + 텍스트 요약 생성
        ↓
html2canvas PNG 내보내기
```

여기서 중요한 특징은 세 가지다.

### 1. 단일 파일 아키텍처

장점:

1. 시작이 빠르다
2. 배포가 단순하다
3. 수정 지점이 직관적이다

단점:

1. 코드가 커질수록 관리가 어렵다
2. 스타일 / 템플릿 / 로직 결합도가 높다
3. 협업 구조로는 불리하다

현재 프로젝트 규모에서는 이 선택이 합리적이다.

### 2. 템플릿보다 데이터 모델이 먼저다

겉보기에는 “이미지 생성기”지만 실제 구현은 먼저 공통 필드를 만들고, 여러 카드가 같은 필드를 소비하는 구조다. 즉, 단순한 정적 템플릿 조합보다 한 단계 더 구조화되어 있다.

### 3. 가변형 비주얼 템플릿 계층이 있다

커버 디자인이 한 종류가 아니다. `crystal`, `cyber`, `ink`, `botanical`, `atelier`, `poster`, `split`, `blueprint`, `brutal`, `luxe`, `neon`, `catalog` 같은 템플릿 계층이 분리되어 있다.

이는 콘텐츠를 지속적으로 생산할 때 반복 인상을 줄이는 데 유효하다.

## 현재 장점

1. 입력부터 결과물까지 기능 폐쇄가 명확하다
2. 실제 콘텐츠 제작 시나리오에 밀착되어 있다
3. AI 사용 방식이 과장되지 않고 실용적이다
4. 시각적 완성도에 신경 쓴 흔적이 있다
5. 배포와 실행이 단순하다

---

## 현재 한계

1. 모든 로직이 하나의 파일에 모여 있어 유지보수성이 낮다
2. DeepSeek와 Reader 같은 외부 서비스 의존도가 있다
3. 현재 기본 설정은 주얼리 디자인 공모전에 치우쳐 있다
4. API Key와 입력 상태에 대한 영속 저장 체계가 약하다
5. 장기적으로는 `schema`, `prompt builder`, `renderer`, `export`, `theme system` 단위로 분리하는 편이 낫다

## 한 줄 평가

`ContestRedbook`은 복잡한 아키텍처를 과시하는 프로젝트가 아니다. 대신 실제 작업 흐름을 정확히 겨냥하고, AI를 필요한 구간에만 배치해서 바로 쓸 수 있는 결과물을 만들어 내는 도구다.

개인 프로젝트 관점에서 보면 단순한 데모를 넘어서, 명확한 사용자와 분명한 산출물을 가진 제품형 프로토타입에 가깝다.
